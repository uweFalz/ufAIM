# Roadmap

This roadmap outlines the planned evolution of ufAIM.
It intentionally focuses on architectural milestones rather than feature lists.

The goal is long-term robustness and engineering validity,
not short-term completeness.

Diese Roadmap beschreibt die geplante Weiterentwicklung von ufAIM.
Sie fokussiert bewusst auf architektonische Meilensteine
und nicht auf kurzfristige Feature-Listen.

Ziel ist langfristige Robustheit und ingenieurtechnische Validität.

Phase 1 — Alignment Core Stabilization
	•	SparseAlignment v1 (fix + transition elements)
	•	Berlin 3pcs transition family
	•	Deterministic sampling and evaluation
	•	Multiband foundation (κ(s) mandatory)

Phase 2 — Import & Semantics
	•	SevenLines workflow stabilization
	•	TRA/GRA semantic interpretation
	•	Validation rules and diagnostics
	•	fat → sparse conversion pipeline

Phase 3 — Multiband Engineering
	•	Gradient and cant integration
	•	Speed, comfort, and travel time bands
	•	Engineering-focused visualization
	•	Band-based validation workflows

Phase 4 — Optimization Readiness
	•	Solver hooks and parameter exposure
	•	Constraint definitions
	•	Initial SQP-based optimization experiments

Phase 5 — BIM Context Integration
	•	IFC as contextual geometry
	•	Alignment overlay and inspection
	•	Deferred IFC alignment authoring/export

Non-Goals
	•	Full CAD replacement
	•	Early IFC authoring
	•	UI-driven design logic
